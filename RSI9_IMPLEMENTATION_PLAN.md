# RSI(9) 単独ロジック 実装検討メモ

## 目的
既存の複合ロジックを停止し、以下の RSI(9) ルールに一本化する。

- Long entry: RSI <= 30
- Long TP: RSI >= 70
- Long SL: RSI <= 20
- Short entry: RSI >= 80
- Short TP: RSI <= 40
- Short SL: RSI >= 90

## 実装方針（段階的）
1. **バックテスト段階**
   - `bars_1m` の close だけで RSI(9) を計算
   - ルール通りのシグナル生成・決済を実装
   - 日次で勝率/損益/DD を出力
2. **ドライラン段階（paper/live off）**
   - 現行監視のループに RSI シグナルを差し込み
   - 既存戦略は `feature flag` で無効化
3. **本番段階（live）**
   - 小ロットで段階導入
   - エントリー失敗時・未約定時の挙動を先に固定

## リスク
- レンジ相場でダマシが連発しやすい
- `RSI<=30` と `RSI>=80` は同日に偏りが出る（片側のみ発火）
- 指値/成行・約定遅延でバックテストとの差分が大きい

## 最低限の追加ガード案
- セッション終端 EOD 強制決済
- エントリー直後の再エントリー禁止クールダウン
- 連敗時の一時停止
